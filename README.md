# Plesk Backup Scheduler

Plesk Backup Scheduler is a lightweight desktop app (PyQt6) that periodically downloads the latest backup archive already generated by Plesk.

It is useful when you want an additional local/off-site copy of Plesk backups on a schedule.

## Features

- GUI-based setup for Plesk URL, credentials, subscription ID, destination folder, and schedule
- Repeating download schedule (8h, 24h, 3d, 1w, 2w, 1m)
- Automatic latest-backup detection
- Resume support for interrupted downloads via `.part` files
- Progress bar with speed and ETA
- CLI helper script for headless/manual usage

## Security Notes

- No credentials are hardcoded in source code.
- Credentials are entered by the user in the GUI or CLI.
- The app stores settings in `config.json` next to the executable/script for convenience.
- `config.json` may contain plain-text credentials. Do not commit it to Git. Protect it at OS level.

## Requirements

- Python 3.10+ (recommended)
- Windows/Linux/macOS supported by Python + PyQt6
- Plesk panel access with permission to view/download backups

Install dependencies:

```powershell
pip install -r requirements.txt
```

## Run (Development)

```powershell
python plesk_backup_scheduler.py
```

## Build EXE (Windows)

```powershell
pip install pyinstaller
pyinstaller --noconfirm --onefile --windowed --name PleskBackupScheduler plesk_backup_scheduler.py
```

Output:

- `dist\PleskBackupScheduler.exe`

You can also build with the spec file:

```powershell
pyinstaller --noconfirm PleskBackupScheduler.spec
```

## How To Use (GUI)

1. Open the app.
2. Enter Plesk connection details:
- Plesk URL, e.g. `https://your-plesk-host:8443`
- Username
- Password
- Subscription (Domain) ID
- Primary domain name (optional, but recommended for fallback parsing)
3. Choose backup destination folder.
4. Select schedule interval.
5. Click Start.

The first download starts immediately, then continues on the selected schedule.

## How To Find Plesk Details

1. Sign in to Plesk as a user with backup access.
2. Open the target subscription/domain.
3. Find the subscription/domain ID in the URL or backup page context.
4. Use your main domain name as optional fallback field.
5. Ensure backups are already being generated by Plesk's backup manager.

## CLI Usage

`download_latest_backup_cli.py` is a command-line alternative.

Example:

```powershell
python download_latest_backup_cli.py --plesk-url https://your-plesk-host:8443 --username your_user --password your_password --domain-id 123 --domain-name example.com --out-dir ./downloads
```

Environment variable alternative:

- `PLESK_URL`
- `PLESK_USER`
- `PLESK_PASSWORD`
- `PLESK_DOMAIN_ID`
- `PLESK_DOMAIN_NAME`
- `PLESK_OUT_DIR`

## Output Naming

Downloaded files are named as:

- `plesk_backup_YYMMDDHHMM.zip`

## Troubleshooting

- Login errors: verify URL, username/password, and account permissions.
- No backups found: create backups first in Plesk.
- HTML/JSON returned instead of ZIP: session may be expired or backup reference invalid.
- TLS warnings: many Plesk setups use self-signed certs; certificate validation is disabled intentionally in this utility.

## Repository Preparation Checklist

Before pushing to public GitHub:

1. Ensure `config.json` is not committed.
2. Ensure build artifacts (`dist/`, `build/`, `.part` files) are not committed.
3. Rotate any previously exposed credentials if they were ever committed.
